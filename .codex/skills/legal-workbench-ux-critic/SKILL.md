# Mission

Critique Debriev UI as a serious legal review workbench with an emphasis on trust, orientation, evidence clarity, and reviewer speed.

# Use when

- reviewing an existing screen
- refining hierarchy or density
- checking whether evidence/provenance is surfaced correctly
- checking whether the interface feels serious or demo-like
- deciding what visual or structural changes matter most

# Do not use when

- no UI exists yet
- the task is backend-only
- the task is just implementing a screen from scratch without critique

# Debriev rules

- This is a review workspace, not a chatbot.
- Main work surface should be obvious immediately.
- Verdict, reason, and next action should be visible quickly.
- Evidence/provenance should feel central, not buried.
- Dense is good if hierarchy remains clear.
- Avoid decorative clutter and fake dashboard filler.

# Review priorities

1. orientation
2. hierarchy
3. evidence clarity
4. decision clarity
5. density and scannability
6. resolved-state usability
7. inspector overload risk

# Inputs

- screenshot or existing component
- current screen purpose
- reviewer task on this screen
- known pain points

# Required output

1. Executive summary
2. What is working
3. What is hurting trust or speed
4. Top hierarchy problems
5. Specific improvements
6. Good enough for now vs must fix soon
7. Optional polish ideas

# Checklist

- Is the main surface obvious in one glance?
- Can I quickly tell what is flagged and why?
- Is the inspector overloaded?
- Is the next action obvious?
- Does this feel like legal infrastructure or a demo?
